# AI广告投放助手 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
AI广告投放助手（AI Ad Assistant）

### 1.2 产品定位
为传统广告投放管理后台提供**AI智能副驾驶**，通过Chrome浏览器插件形式，实时分析页面信息，积累行业优化知识，在适当时机向运营人员提供数据预警、效率优化和策略建议。

### 1.3 核心价值主张
- **零侵入**：不修改现有后台代码，插件独立运行
- **AI自适应**：由DeepSeek动态理解任意广告后台页面结构，不硬编码选择器
- **纯本地**：所有持久化数据存储在用户本地浏览器IndexedDB中
- **可迁移**：通过导入导出功能实现多设备数据同步

### 1.4 目标用户
广告投放运营人员（内部工具），需要日常监控和管理多个广告平台的投放数据。

## 2. 用户故事

### US-01: 页面数据感知
> **作为**广告运营人员，**我想要**打开任意广告后台页面时插件能自动识别页面类型和数据结构，**以便**我不需要手动配置就能获取AI辅助。

**验收标准**：
- 插件加载后3秒内完成页面DOM快照采集
- DeepSeek AI在5秒内返回页面理解结果
- 支持的页面类型：报表(report)、计划列表(campaign_list)、广告创意(ad_creative)、设置(settings)
- 首次识别后缓存页面模板，同结构页面复用缓存

### US-02: 数据异常预警
> **作为**广告运营人员，**我想要**在关键指标出现异常时收到及时预警，**以便**我能快速响应避免损失。

**验收标准**：
- 自动检测以下异常类型：
  - CTR异常下降（低于历史均值30%）
  - 消耗异常（超预算风险、消耗停滞）
  - CPA异常波动
  - 转化量骤减
- 异常以侧边栏面板形式展示，数据异常行高亮标记
- 支持查看异常详情和忽略操作

### US-03: 效率优化建议
> **作为**广告运营人员，**我想要**系统能检测我的重复操作并给出效率优化建议，**以便**减少日常重复劳动。

**验收标准**：
- 检测重复性操作模式
- 推荐更高效的筛选和查看方式
- 自动保存常用筛选条件
- 异常数据自动置顶

### US-04: 策略建议
> **作为**广告运营人员，**我想要**基于历史数据获得投放策略优化建议，**以便**提升广告投放ROI。

**验收标准**：
- 低CTR计划：建议优化创意方向
- 高CPA计划：建议调整出价或人群定向
- 预算分配建议
- 从累积数据中自动发现优化模式

### US-05: 数据管理与迁移
> **作为**广告运营人员，**我想要**能够导入导出我的数据，**以便**在不同设备间同步使用。

**验收标准**：
- 支持导出记忆数据生成`.admemory`文件
- 支持导入`.admemory`文件并智能合并排重
- 导出文件带SHA-256校验和
- 支持选择导出范围（全部/仅配置/仅业务数据）
- 提供多种合并策略（自动合并/逐项确认）

### US-06: 用户偏好设置
> **作为**广告运营人员，**我想要**自定义插件的建议接收偏好，**以便**获得符合我工作节奏的辅助。

**验收标准**：
- API Key配置与连接测试
- 建议总开关 + 分类开关（异常/效率/策略）
- 触发频率设置（实时/5分钟/15分钟/手动）
- 敏感度阈值调整
- 设置持久化保存

## 5. MVP范围定义

### 第一阶段：基础感知 + 数据异常预警
1. AI自适应页面理解（报表页面优先）
2. 业务数据自动提取
3. 数据异常实时检测（CTR/消耗/CPA/转化量）
4. 异常预警侧边栏面板
5. 本地数据持久化
6. API Key配置

### 第二阶段（后续迭代）
1. 效率优化建议
2. 策略建议
3. 知识自动挖掘
4. 云端同步

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| DeepSeek API不稳定 | 核心功能不可用 | 本地缓存+重试机制+降级提示 |
| 页面结构变化频繁 | 识别准确率下降 | 模板失效检测+自动重新识别 |
| IndexedDB容量限制 | 历史数据丢失 | 定期导出提醒+数据修剪策略 |
| 浏览器插件审核 | 上线延迟 | 遵循Chrome Web Store规范 |
